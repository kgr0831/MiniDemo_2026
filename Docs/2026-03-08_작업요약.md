### 3. 스태미나 UI Bloom 효과 (Post-Processing)
**가장 큰 작업.** 스태미나 점멸 시 진짜 빛나는 Bloom 효과를 적용하기 위해 많은 시행착오를 거쳤습니다.

#### 시도했던 방법들과 실패 원인

| 시도 | 방법 | 실패 원인 |
|------|------|-----------|
| 1 | `Image.color`에 HDR 값(4.0~30.0) 직접 대입 | `CanvasRenderer`가 색상을 0~1로 강제 클램핑 |
| 2 | Sprite-Lit-Default 쉐이더 | 조명을 받아야만 밝아지는 쉐이더라 UI에서 발광 불가 |
| 3 | Sprite-Unlit-Default 쉐이더 | HDR 값이 여전히 클램핑됨 |
| 4 | 커스텀 쉐이더 (`fixed4` 타입) | `fixed4`가 0~1로 클램핑하는 저정밀도 타입 |
| 5 | 커스텀 쉐이더 (`half4`로 수정) | UI Canvas가 Post-Processing **이후에** 렌더링되어 Bloom 적용 불가 |
| 6 | `material.SetFloat("_Intensity")` 방식 | 쉐이더 내부에서 곱하지만, Canvas 렌더 순서 문제 동일 |
| 7 | UIEffect 에셋 (mob-sakai) `samplingIntensity` | 블러만 적용되고 Bloom 같은 발광 효과 아님 |
| 8 | UIEffect `Shadow + Glow 모드` | 후광이 생기긴 하나 Bloom과는 다른 느낌 |

#### 최종 해결: Overlay Camera + 커스텀 쉐이더

**핵심 원인:** Main Camera의 `HDR Rendering`이 `Off`로 되어 있어서, 쉐이더가 아무리 HDR 값을 출력해도 카메라 렌더 버퍼가 0~1로 잘라버리고 있었음.

**최종 구성:**
1. **UI 전용 Overlay Camera** 생성 → Main Camera의 Stack에 추가
2. Canvas의 Render Camera를 UI Camera로 연결
3. Main Camera의 `HDR Rendering` → `Use settings from Render Pipeline Asset`로 변경
4. URP Asset의 `Quality > HDR` 체크 확인
5. URP Asset의 `Post-processing > Grading Mode` → `High Dynamic Range`
6. **UI_Glow.shader**: `_Intensity` 프로퍼티로 RGB를 곱하는 커스텀 UI 쉐이더
7. **StaminaBarUI.cs**: `material.SetFloat("_Intensity", 5.0f)`로 쉐이더에 직접 HDR 강도 전달
8. Global Volume Bloom `Threshold: 1.2` → 평소(1.0)는 안 빛나고, 점멸 시(5.0)만 Bloom 발동
